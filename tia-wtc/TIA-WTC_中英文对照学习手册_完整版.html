1. Preamble | 前言

文档概述与用途说明
Document Purpose | 文档用途:
This description is for experienced Mine Winder engineers only. It does not describe a mine winder or the TIA-WTC, but it shows how the system is structured.

The TIA-WTC is designed not only to fulfill the requirements of mining authorities but also to get a Safety Integrity Level 3 (SIL3). For SIL3 purposes some special functions are implemented.
中文翻译：
本说明书仅适用于有经验的矿井提升机工程师。它不描述矿井提升机或TIA-WTC本身，而是展示系统的结构组成。

TIA-WTC的设计不仅满足矿山监管机构的要求，还达到了安全完整性等级3（SIL3）。为了实现SIL3，实施了一些特殊功能。
💡 关键信息：
• 目标读者：经验丰富的提升机工程师
• 文档性质：系统结构说明，非操作手册
• 安全等级：SIL3（安全完整性等级3）
• 版本：Rev. 05 (31.07.2021)

2.1 Control System | 控制系统

TIA-WTC硬件架构与通道配置
System Architecture | 系统架构:
The TIA-WTC is built as a double channel system with an S7-1516 in Channel A and S7-1515 in Channel B. Additionally there is a Simotion D445 which belongs to Channel A. For system supervisions a Fail-safe CPU necessary, therefore, a S7-1214F is installed. If SIL3 is requested the Fail-safe CPU also takes care of the shaft limit switches.
中文翻译：
TIA-WTC构建为双通道系统，通道A使用S7-1516，通道B使用S7-1515。此外，还有一个属于通道A的Simotion D445。为了进行系统监控，需要故障安全CPU，因此安装了S7-1214F。如果要求SIL3，故障安全CPU还负责井筒限位开关。

⚠️ 重要说明：
• 通道A和B使用不同类型的CPU，这是出于SIL3/TAS要求的考虑
• 通道A和B使用不同类型的编码器输入，符合SIL3要求
• 使用"小型"CPU以获得更好的MTBF（平均故障间隔时间）值
• 每个通道最多可读取6个HTL脉冲编码器和16个同步开关

2.2 Data Recorder (Option) | 数据记录仪（可选）

数据采集与记录系统
Hardware Configuration | 硬件配置:
The datarecorder is a Simatic BOX PC with ibaPDA running on it. The data from the control system can be received via ibaBM-DP (Profibus) or ibaBM-PN (Profinet). The standard configuration is Profibus, and it reads signals from the S7-1516 Channel A, the Simotion and the drive.
中文翻译：
数据记录仪是一台运行ibaPDA的Simatic BOX PC。来自控制系统的数据可以通过ibaBM-DP（Profibus）或ibaBM-PN（Profinet）接收。标准配置是Profibus，它读取来自S7-1516通道A、Simotion和传动装置的数据。
📋 功能特性
• The datarecorder can be operated via AMT over Ethernet from the Diagnostic Notebook
• 数据记录仪可以通过以太网从诊断笔记本通过AMT操作
• Additionally, in the Diagnostic Notebook a screen is available with trigger settings and display of the system status
• 此外，在诊断笔记本中有一个屏幕，可用于触发设置和显示系统状态
• In order to get signals from the S7-1515 Channel B it could be extended (additional hardware and software licenses required)
• 为了从S7-1515通道B获取信号，可以扩展（需要额外的硬件和软件许可证）

2.6 Failsafe CPU | 故障安全CPU

SIL3和TAS等级实现
Safety Supervision | 安全监控:
In order to reach a SIL3 and TAS level it is necessary to supervise the standard CPUs (S7-1500) via a failsafe CPU (S7-1200F). It is sufficient to supervise the CPU in Channel A only. Therefor a special software is running in the supervisions of Channel A and three bits are send to the S7-1200F.
中文翻译：
为了达到SIL3和TAS等级，必须通过故障安全CPU（S7-1200F）来监控标准CPU（S7-1500）。仅监控通道A中的CPU就足够了。因此，在通道A的监控功能中运行特殊软件，并将三个位发送到S7-1200F。
⚠️ 安全机制：
• 如果这三个位有任何错误，S7-1200F将切断通道A和B的安全电路继电器
• 如果要求SIL3，S7-1200F还必须负责井筒限位开关

HMI PC是装有TIA-WinCC的Simatic Microbox PC。它们仅通过Profinet连接到通道A和B中的控制系统CPU。在标准配置中，可以将3台PC和1个操作面板连接到TIA-WTC。

2.9 Bus Systems | 总线系统

通信系统架构
Interface Configuration | 接口配置:
The Interface between: Channel A and B is Profinet IO with Channel A as master. Channel A/B and HMI/Diagnostic/Engineering Notebook is Profinet. Channel A and Simotion is Profibus DP with Channel A S7-CPU as master.
中文翻译：
接口之间：通道A和B是Profinet IO，通道A为主站。通道A/B和HMI/诊断/工程笔记本是Profinet。通道A和Simotion是Profibus DP，通道A S7-CPU为主站。
通信接口 (Interface) 总线类型 (Bus Type) 主从关系 (Master-Slave)
Channel A ↔ Channel B Profinet IO Channel A = Master
Channel A/B ↔ HMI/Diagnostic Profinet -
Channel A ↔ Simotion Profibus DP Channel A = Master
Simotion ↔ Drive/Datarecorder Profibus or Profinet Simotion = Master
Channel A/B ↔ Machine Control Periphery Profibus or Profinet Standard: Profibus

以太网通信核心组件
Switch Specification | 交换机规格:
All Ethernet and Profinet communication run via one Scalance switch in the TIA-WTC. It is a 12-port switch with one additional fiber optic connection (not used in standard configuration).
中文翻译：
TIA-WTC中的所有以太网和Profinet通信都通过一个Scalance交换机运行。这是一个12端口交换机，带有一个额外的光纤连接（标准配置中未使用）。

2.11 Time Synchronization | 时间同步

系统时间同步机制
Synchronization Method | 同步方法:
The CPU Channel A is time master. It can be set manually from an HMI-PCs or from the Diagnostic Notebook. Additionally, there is a chance of setting the time via a DCF77-signal. The time synchronization is done via the CPUs "Local time".
中文翻译：
CPU通道A是时间主站。可以手动从HMI-PC或诊断笔记本设置。此外，可以通过DCF77信号设置时间。时间同步通过CPU的"本地时间"完成。
设备 (Device) 同步方式 (Synchronization Method)
CPU Channel B NTP (网络时间协议)
Simotion Channel A Profibus user data
Brake System Profibus/Profinet user data
HMI Devices & Diagnostic App "Area Pointers"
ibaPDA-System (可选) "ibaBM-DP" or "ibaBM-PN"
S7-1200F 不需要同步
Data concentrator (可选) Profibus user data

2.12 Data Concentrator (Option) | 数据集中器（可选）

外部接口与OPC UA服务器
Function | 功能:
The data concentrator is a S7-1211C-CPU and serves as interface to extern (Customer, Winder App, etc.). The data concentrator is connected to the S7-1500 CPU channel A via Profibus. Profibus as interface assures that there is no possibility to get any access from extern to the TIA-WTC Channel A/B.
中文翻译：
数据集中器是S7-1211C-CPU，用作对外部（客户、提升机应用等）的接口。数据集中器通过Profibus连接到S7-1500 CPU通道A。Profibus作为接口确保不可能从外部访问TIA-WTC通道A/B。
💡 安全特性：
• 使用Profibus接口，确保外部无法访问TIA-WTC内部网络
• 在数据集中器上运行OPC UA服务器
• 外部只能通过以太网使用OPC UA访问数据集中器

3.1.4.1 Safety Circuits | 安全电路

安全电路合并机制
Circuit Structure | 电路结构:
The three Safety Circuits (LOC, ESC, SAC) are split into three parts. Each of the three main software modules (supervisions, machine control, shaft signaling system) has its own separate three circuits. The three safety circuits of the three main modules are merged to three circuits (LOC, ESC, SAC) in the system function "Safety Circuits".
中文翻译：
三个安全电路（LOC、ESC、SAC）分为三个部分。三个主要软件模块（监控功能、机器控制、井筒信号系统）每个都有自己独立的三个电路。三个主要模块的三个安全电路在系统功能"Safety Circuits"中合并为三个电路（LOC、ESC、SAC）。
💡 安全电路说明：
• LOC (Local Safety Circuit) - 本地安全电路
• ESC (Emergency Stop Circuit) - 紧急停止电路
• SAC (Safety Application Circuit) - 安全应用电路

3.1.4.2 Safety Concept | 安全概念

安全概念合并机制
Concept Structure | 概念结构:
The Safety Concept is split into three parts. Each of the three main software modules (supervisions, machine control, shaft signaling system) has its own separate safety concept. The safety concept of the three main modules is merged to one safety concept in the system function "Safety Concept".
中文翻译：
安全概念分为三个部分。三个主要软件模块（监控功能、机器控制、井筒信号系统）每个都有自己独立的安全概念。三个主要模块的安全概念在系统功能"Safety Concept"中合并为一个安全概念。
